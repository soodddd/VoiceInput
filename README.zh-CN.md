# VoiceInput

**v0.1.2-preview · Windows 本地语音输入**

[English](./README.md) | [简体中文](./README.zh-CN.md)

> ⚠️ **预览版本（Preview）** — 这是用于测试和反馈的早期预览版本，正式版发布前可能有 bug 和变动。

一个注重隐私的、完全本地运行的 Windows 语音输入工具。对着麦克风说话，你的话就会自动出现在光标所在的位置——不需要打字。语音识别完全在你的显卡上通过 [Qwen3-ASR](https://github.com/QwenLM/Qwen3) 完成，**你的语音数据绝对不会离开你的电脑**。

---

## VoiceInput 是什么？（给小朋友也能看懂的版本）

VoiceInput 就像一个帮你打字的小助手。你不用自己敲键盘，只需要：

1. **按住键盘上的 `Alt+V` 键**不放
2. 对着麦克风**说话**
3. **松开** `Alt+V` 键
4. 你说的话就会**自动变成文字**，出现在光标闪动的地方！

它在**任何能打字的地方都能用**——Word、微信聊天框、浏览器搜索框、游戏里，到处都行。而且因为它完全在你自己的电脑上运行，没有任何人能听到你的录音。

---

## 功能特性

- 🎙️ **按住说话** — 按住 `Alt+V` 说话，松开自动识别并粘贴
- 🖱️ **点击说话** — 点击麦克风按钮开始，再点击停止
- 🌐 **语言切换** — `Alt+L` 在 自动/中文/英文 之间切换
- 🔒 **100% 离线** — 所有识别在本地显卡完成，零云端调用
- 📝 **自定义术语** — 设置"识别错误→正确文字"的纠正规则
- 🩺 **自动恢复** — 后端崩溃自动重启 + 文件日志
- 🖥️ **系统托盘** — 快速访问设置、日志和退出
- ⚡ **VAD 静音停止** — 检测到静音自动停止录音
- 📋 **智能粘贴** — 自动粘贴到光标位置，之后恢复你原来的剪贴板
- 🔄 **更新检查** — 有新版本时自动提醒你

## 系统要求

| 要求 | 详情 |
|------|------|
| 操作系统 | Windows 10 1903+ / Windows 11 |
| 显卡 | **NVIDIA** 显卡，支持 CUDA，显存 ≥ 4 GB |

## 快速开始（一步一步教你用）

### 第一步：下载

1. 打开 [发布页面](../../releases)
2. 下载名字叫 `VoiceInput-v0.1.2-preview-win64.zip` 的文件

### 第二步：解压

1. 找到下载的 `.zip` 文件（一般在"下载"文件夹里）
2. **右键点击**这个 zip 文件 → 选择 **"全部解压缩..."**
3. 选一个保存位置，比如：`C:\VoiceInput`
4. 点击 **"提取"**（或"解压"）

> 💡 **小提示：** 不要放在需要管理员权限的文件夹里（比如 `C:\Program Files\`），否则可能会出问题。

### 第三步：运行

1. 打开解压后的文件夹
2. **双击** `voiceinput.exe`
3. 屏幕上会出现一个小小的悬浮窗口

> 🛡️ 如果 Windows 弹出"Windows 已保护你的电脑"的提示，点击 **"更多信息"** → **"仍要运行"**。这对不在微软商店里的软件很常见，不用担心。

### 第四步：下载模型（只有第一次需要）

第一次运行 VoiceInput 时，需要下载 AI 模型（大约 1.2 GB）。**这只需要做一次**。

1. 程序会显示下载界面
2. 选择 **ModelScope**（推荐国内用户，速度快）或 **HuggingFace**（国际用户）
3. 点击 **"开始下载"**
4. 等待下载完成（取决于网速，一般 5-20 分钟）
5. 下载完成后点击 **"加载模型"** — 大约需要 10-20 秒

### 第五步：开始说话！

1. 打开任何能打字的程序（比如 Word 文档、微信聊天框、浏览器搜索框）
2. 把光标点击到你想让文字出现的位置
3. **按住键盘上的 `Alt+V` 键**
4. 对着麦克风**说话**
5. **松开 `Alt+V` 键**
6. 你说的话就会自动变成文字出现啦！

## 使用方法

### 悬浮窗介绍

屏幕上的小窗口有这些部分：

- **麦克风按钮**（大圆圈）：点击开始录音，再点击停止
- **语言标志**（左上角，显示"Auto"/"中"/"EN"）：点击切换语言
- **齿轮图标**（右上角）：点击打开设置
- **X 图标**（右上角）：点击隐藏窗口（程序仍在系统托盘运行）

### 键盘快捷键

| 快捷键 | 作用 |
|--------|------|
| `Alt+V`（按住） | 按住说话，松开识别 |
| `Alt+L` | 切换语言：自动 → 中文 → 英文 → 自动 |

### 系统托盘

屏幕右下角（时间旁边）会出现一个绿色的麦克风图标。**右键点击**它可以：
- 显示/隐藏面板
- 打开设置
- 切换语言
- 加载/释放模型（释放显存）
- 查看日志文件
- 退出程序

## 设置说明

点击悬浮窗上的**齿轮图标**，或右键托盘图标 → **设置**，可以打开设置面板。

### 麦克风 标签页
选择使用哪个麦克风。点击**测试**按钮可以录一小段试试看麦克风好不好使。

### 快捷键 标签页
修改录音快捷键（默认 Alt+V）和语言切换快捷键（默认 Alt+L）。

### 音频 标签页
调整采样率、音频归一化、裁剪静音、静音阈值。

### 高级 标签页
- **模型策略**：AI 模型如何使用显存
  - **平衡模式**（默认）：空闲 30 分钟后自动释放
  - **性能优先**：始终加载（响应最快）
  - **省显存**：每次识别后释放（显存占用最低）
  - **质量优先**：最大精度
- **标点模式**：标点符号处理方式
  - **简单标点**（默认）：自动在末尾加句号
  - **原始输出**：不做处理
  - **输入法模式**：去掉末尾标点（适合搜索框）
- **中英自动空格**：在中文和英文之间自动加空格
- **VAD 静音检测**：检测到 2 秒静音后自动停止录音
- **开机自启**：开机后自动启动 VoiceInput
- 粘贴延迟、恢复剪贴板、最大录音时长、请求超时

### 术语 标签页
添加自定义词语纠正。比如如果 AI 总是把你的名字"小明"识别成"小名"，可以加一条规则纠正它。

## 常见问题

### 程序打不开 / 打开马上就关了
- **最可能的原因：** 没有 NVIDIA 显卡。VoiceInput 必须有 NVIDIA 显卡才能运行。按 `Ctrl+Shift+Esc` 打开任务管理器 → "性能"标签页，看看有没有"NVIDIA"。
- **解决办法：** 安装最新的 [NVIDIA 驱动](https://www.nvidia.com/Download/index.aspx)。如果是双显卡笔记本，在 NVIDIA 控制面板里设置 VoiceInput 使用独立显卡。

### 提示"已有另一个实例在运行"
- VoiceInput 已经打开了。在屏幕右下角（时间旁边）找绿色的麦克风图标。

### 录音开始了但是没有文字出现
- **检查：** 模型加载了吗？打开设置 → 高级 → 模型策略。如果显示未加载，右键托盘图标 → **预加载模型**。
- **检查：** 麦克风正常吗？打开设置 → 麦克风 → 测试。

### 文字出现在了错误的位置
- VoiceInput 会把文字粘贴到**光标所在的位置**。说话前先点击你想让文字出现的地方。
- 有些程序（比如某些游戏）可能阻止粘贴，试试先点击在文本框里。

### 识别很慢
- 加载后的第一次识别会比较慢（模型在预热），之后会变快。
- 试试在设置里把模型策略改成**性能优先**（始终加载）。
- 确保没有运行其他吃显存的程序。

### 下载失败 / 下载很慢
- 国内用户选择 **ModelScope**（快），国际用户选择 **HuggingFace**。
- 下载支持断点续传——如果中断了，再次点击下载会继续。

## 从源码构建

### 前置要求

- [Node.js](https://nodejs.org/) 18+ 和 npm
- [Rust](https://rustup.rs/)（stable 工具链）
- Python 3.10+ 及后端依赖（`torch`、`fastapi`、`qwen_asr` 等）
- [Tauri CLI 2.x](https://tauri.app/)（`npm install -D @tauri-apps/cli`）
- NVIDIA CUDA Toolkit 11.8+

### 步骤

```bash
# 1. 安装前端依赖
npm install

# 2. 构建 Python ASR 后端（PyInstaller onefile → src-tauri/binaries/）
.\build_backend.bat

# 3. 开发模式运行
npm run tauri dev

# 4. 构建发布 zip（voiceinput.exe + sidecar + 资源）
powershell -ExecutionPolicy Bypass -File .\build_release_zip.ps1
```

发布的 zip 输出到 `.\release\VoiceInput-v0.1.2-preview-win64.zip`。

> **关于 NSIS** — PyInstaller onefile 打包的 sidecar 约 2.7 GB（包含 torch + transformers），超过了 NSIS 的 mmap 限制。项目改用 zip 分发方案。详见 `build_release_zip.ps1`。

## 架构

```
voiceinput.exe (Tauri/Rust)  ──启动──▶  asr_backend.exe (Python/FastAPI)
        │                                        │
        ├─ cpal（录音）                          ├─ Qwen3-ASR-0.6B（GPU 推理）
        ├─ enigo（粘贴模拟）                     ├─ uvicorn（HTTP 服务 127.0.0.1:8765）
        ├─ rdev（全局快捷键）                    └─ postprocess（术语纠正）
        └─ React 18（悬浮窗 UI）
```

| 层级 | 技术 | 关键文件 |
|------|------|----------|
| 前端 | React 18 + TypeScript + Tailwind | `src/` |
| 原生外壳 | Rust + Tauri 2 | `src-tauri/src/` |
| ASR 后端 | Python + FastAPI + PyTorch | `backend/` |

## 配置文件

用户配置在 `%LOCALAPPDATA%\VoiceInput\config.json`。日志在 `%LOCALAPPDATA%\VoiceInput\logs\`。
